Data Flow

  1. Users authenticate to the system through Google OAuth.
  2. Frontend Vue.js application communicates with the Flask backend API via HTTP/REST endpoints.
  3. Backend Flask application stores and retrieves data from the PostgreSQL database using SQLAlchemy ORM.
  4. GPS tracking devices (both physical and simulated) connect to the Protocol Server via TCP on port 8080.
  5. Protocol Server automatically detects the protocol type (808 or JT808) and parses the incoming messages.
  6. Protocol Server processes messages containing location updates, health data, and device status.
  7. Protocol Server forwards processed data to the Backend for storage and further processing.
  8. Location Service processes geographical data, calculates distances, and detects geofence violations.
  9. Backend delivers real-time location and status updates to the Frontend.
  10. Frontend displays the pet locations, tracks, and status on maps and dashboards.

Protocol-Specific Extensions

The system supports pet-specific extensions to both protocols:

  • Battery level monitoring (percentage)
  • Pet activity level tracking (percentage of movement)
  • Health status flags (temperature warning, inactivity, etc.)
  • Temperature readings (for health monitoring)

Environment Configuration

The system relies on several environment variables for configuration:

Backend Environment

  • DATABASE_URL: PostgreSQL connection string
  • SESSION_SECRET: Session encryption key
  • GOOGLE_OAUTH_CLIENT_ID: OAuth client ID
  • GOOGLE_OAUTH_CLIENT_SECRET: OAuth client secret
  • PROTOCOL_808_PORT: Protocol server port (default: 8080)

Frontend Environment

  • VITE_API_BASE_URL: Backend API URL (default: http://localhost:5000)

The VITE_API_BASE_URL is critical as it determines where the frontend sends API requests. It is used in the api.js service to construct all API endpoints.

Development and Testing

The Device Simulator tool allows testing the system without physical devices:

python tools/device_simulator.py --device-id test123 --imei 123456789012345 --interval 5 --protocol jt808
